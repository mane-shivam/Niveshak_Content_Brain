## STATUS DEFINITIONS

**STRONG**: Framework applications consistently validated by market reality  
**WEAK**: Some applications fail; thesis has holes  
**FAILED**: Framework broken by market events; needs retirement  
**UNTESTED**: Too new to evaluate (<3 uses)

---

## WIN RATE CALCULATION

**Win** = Framework thesis held up; no contradiction from market/data  
**Loss** = Framework thesis broken; had to issue correction or thesis invalidated  
**Neutral** = Framework used for teaching only (no testable thesis)

**Win Rate** = Wins / (Wins + Losses) Ã— 100%

**Target**: >80% win rate to stay STRONG

## RETIREMENT CRITERIA

**Retire Framework If**:
- Win rate <50% over >10 uses
- Systematically broken by market regime change
- Better replacement framework developed
- No reinforcement possible for 6+ months (topic became irrelevant)

**Retirement Process**:
1. Document failure pattern
2. Extract learning
3. Update frameworks_index.md status to RETIRED
4. Remove from active prompts
5. Archive in retired_frameworks.md
